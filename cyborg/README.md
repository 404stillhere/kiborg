# Cyborg — бета оболочки-оркестратора киборга

Первая рабочая **оболочка**, что собирает органы в одного агента. Построена по вердикту
совета 5 моделей (DeepSeek/ChatGPT/Mistral/Qwen/Z.ai, 2026-07-11): после сбора органов
следующий шаг — не 48-й орган, а **оркестратор**.

## Архитектура (агентный цикл)
```
цель → РОУТЕР (отбирает подмножество органов) → МОЗГ (выбирает следующий)
     → ИСПОЛНИТЕЛЬ (безопасно вызывает) → ПАМЯТЬ (env.memory) → повтор → результат
```
- **`router.py`** — ключевой инсайт совета: НЕ отдавать мозгу все органы разом (паралич
  выбора/галлюцинации/переполнение). По цели отбирает k релевантных (счёт по словам). Без LLM.
- **`brain.py`** — планировщик. `env['llm']` есть → LLM-мозг (в проде ask_llm с ключом);
  нет → детерминированный stub-планировщик по дата-флоу (реально работает без ключа).
- **`executor.py`** — прод-гейт: органы, что пишут в живой прод или требуют отсутствующий
  ключ, в safe_mode НЕ запускаются (skipped). Падение органа не роняет цикл (error → перепланирование).
- **`core.py`** — `Organ` (обёртка run(inputs,env)+метаданные) и `Memory` (env.memory +
  `blocked`: упавшие/гейтнутые органы не переизбираются).
- **`registry.py`** — каталог `_shared/organs.json` (89 карточек), «книга инструментов».
- **`wiring.py`** — ИСПОЛНЯЕМЫЕ органы беты: **7** (локальны, чисты, безопасны, растим по одному).
  Дорожка идей: `collect_source → ideate → rank_ideas (судья) → scrub_secrets → deliver (инбокс)`.
  `collect_source` тянет сырьё из 5 лент, смешивая за прогон: Hacker News, Reddit (r/SideProject),
  Lobsters, GitHub Trending (все публичные, без ключа) + Telegram-каналы (личный аккаунт — единственная
  лента с кредами). Упавшая лента прогон не рвёт: берётся из остальных, ошибка в `partial_errors`.
  Дорожка «доделать»: `finish_step → finish_sink`. Плюс `build_harvest_organs()` — автономный
  накопитель: та же цепочка идей, но сток `stash_ideas` (копилка без потолка вместо инбокса).
- **`orchestrator.py`** — сам цикл (`Cyborg.run(goal, env)`).

## Запуск

## Что работает / что заглушка (честно)
- **Работает:** цикл, роутер, дата-флоу планировщик, прод-гейт, память, перепланирование
  при падении. На цели «приноси идеи» киборг сам ведёт цепочку: роутер → collect_source →
  ideate (6 кандидатов) → rank_ideas (судья отбирает топ-3) → scrub_secrets → deliver в инбокс
  (потолок 3, обратная тяга, дедуп уже предложенного). На «доделать» → finish_step (ротация по
  проектам) → finish_sink в отдельный слот. Каждый прогон пишется в `data/runs.md`.
- **Ключ Gemini вживлён:** идеи и рубрика судьи считаются живой моделью (`ask_llm`,
  gemini-2.5-flash). Нет ключа — `ideate`/`rank_ideas` тихо падают на stub-качество (без ошибки).
- **Заглушка осознанно:** мозг-планировщик = детерминированный stub (не LLM) — для линейного
  конвейера надёжнее; LLM-мозг включим, когда органов станет много. Исполняемых органов 7 (каталог — 89).

## Прод НЕ трогается
Оркестратор только читает реестр + зовёт органы через их контракт `run(inputs,env)`
(органы чисты, без сайд-эффектов на импорт). Прод-органы в safe_mode не запускаются.
Всё — только новые файлы под `M:/projects/kiborg/cyborg/`.

## Известные пределы беты (осознанные, не баги — по ревью скептиков)
- **«produced» — навсегда в рамках одного run:** орган, записавший свой ключ (даже пустым),
  повторно не вызывается. Это рвёт холостой спин, но исключает ретрай с бэкоффом (напр.
  временный сбой сети, что прошёл бы на след. шаге). Понадобится — ввести флаг `retryable`.
- **Роутер при 0 совпадений отдаёт мозгу ВСЕ органы** — при большом наборе исполняемых
  органов вернётся «паралич выбора», от которого и защищались. Пока исполняемых 7 — ок;
  пересмотреть, когда набор вырастет (тогда — иерархия/эмбеддинги на отбор).
- **Роутер глух к словоформам** — счёт по точному совпадению слов, склонения не матчатся.
  «доделать существующие проекты» работает, «доделай существующий проект» даёт 0 → пустой цикл.
  Поэтому примеры целей выше — в рабочей форме. Фикс (стемминг/словоформы/LLM-роутер) — когда снимут заморозку ядра.
