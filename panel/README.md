# panel — пульт киборга

Живой UI над киборгом: показывает его «тело» (органы, сердце-инбокс, глаза-источники),
статус источников 🟢/🔴, инбокс идей — и управляет прогонами кнопками. Чистый stdlib-сервер
(`http.server`), без фреймворков. Панель только ЧИТАЕТ код киборга + шлёт 2 действия; сам
`cyborg/`/`idea_engine/` не меняет.

## Запуск

```bash
M:/projects/funpay/venv/Scripts/python.exe panel/serve.py   # → http://127.0.0.1:8737
```

Правил серверную часть (`serve.py`) — перезапусти пульт; правки `index.html`/`bodies.js`
подхватываются обновлением страницы.

## Файлы

| Файл | Что |
|---|---|
| `serve.py` | сервер (`ThreadingHTTPServer`, порт 8737): API + отдача статики + автоцикл |
| `index.html` | весь UI (разметка + CSS + JS) |
| `bodies.js` | SVG-тела облика рыцаря (3 стиля: metal/flat/poly) |

## API

| Метод, путь | Что делает |
|---|---|
| `GET /` · `/bodies.js` | страница и SVG-тела |
| `GET /api/state` | снимок состояния: инбокс, журнал прогонов, источники, автономность, каталог органов, направление идей, ленты-источник, папки-источник |
| `POST /api/run` | «принеси идеи» — прогон `run.py` (роутер выбирает органы под цель) |
| `POST /api/observe` | «наблюдать» — `observe_sources.py`, обход источников от первого лица |
| `POST /api/auto` | включить/выключить автономность + интервал (мин); фон живёт в пульте |
| `POST /api/direction` | руль темы идей: `current` (строка, «» = снять) и/или `presets` (список) |
| `POST /api/feeds` | ленты-источник: `enabled` (список имён лент) — какие публичные ленты включены (тумблеры); канон-порядок/дедуп/только-известные на сервере |
| `POST /api/folders` | папки-источник: `paths` (список путей) — какие папки читать как сырьё; в ответе `probe` (путь валиден? сколько файлов?) |
| `GET /api/folders/probe` | проба текущих папок: по каждому пути — существует ли + сколько пригодных текстовых файлов (счётчики пульта при загрузке) |
| `POST /api/stop` | оборвать текущий прогон |
| `POST /api/idea` | статус идее: `take` / `later` / `trash` |

Прогоны (`run.py`/`harvest.py`/`observe_sources.py`) запускаются отдельным подпроцессом
(`_start_proc`), их stdout стримится в живую консоль страницы построчно. Автономный фон
крутит `harvest.py` по интервалу — **умрёт пульт, встанет фон**.

**Живой прогресс прогона** (конвейер с реальными моделями идёт минуты — без обратной связи казался
зависшим): `run.py` шлёт с `flush` строки «⏳ иду: <орган>» / «✓ готов» (колбэк `orchestrator.on_step`),
а медленные органы — суб-шаги «генерирую N идей» / «читаемость: переписываю i/N» (env-контракт
`on_progress`). Пульт (`pollRun`) рисует их в консоли и подсвечивает РЕАЛЬНО текущий орган на каркасе-рыцаре.
