# Stress tests — нагрузочные тесты kiborg

Эта папка содержит нагрузочные тесты, которые **НЕ запускаются автоматически в CI** (находятся вне per-package `tests/` директорий). Они предназначены для локального прогона перед релизом или при подозрении на утечки памяти/деградацию производительности.

## stress_test_harvest.py

### Что делает

- Запускает `N` прогонов harvest (по умолчанию 50) с моками:
  - **LLM отключен** (`ask_llm.available = False`) → органы на стабах
  - **Сеть отключена** (`collect_source.run` возвращает 3 фейковых items)
  - **Файлы редиректятся** во временную директорию (`state.json`, `runs.md`, `seen_items.json`)
- Замеряет:
  - Среднее время прогона (мс/итерация)
  - Пиковый рост памяти (KB)
  - Наличие ошибок в `runs.md` (должно быть 0)

### Как запускать

```bash
# По умолчанию 50 прогонов
python stress/stress_test_harvest.py

# Указать число прогонов (например, 10 для быстрого теста)
python stress/stress_test_harvest.py 10

# Максимальный потолок (1000) — для экстремального теста
python stress/stress_test_harvest.py 1000
```

### Что ожидается

После прогона N=50 (на типичной dev-машине):
- Среднее время: **~200-500ms/итерация** (зависит от CPU)
- Пиковый рост памяти: **<5000KB** (без утечек)
- Ошибок в runs.md: **0**

Если ошибки > 0 — смотри временный runs.md (выводится в лог).

### Интеграция с RUNBOOK.md

После первого прогона с N=50 добавь результат в `RUNBOOK.md` секцию «Стресс-тест»:

```markdown
**Стресс-тест (прошёл перед релизом):**
- 50 прогонов, среднее 350ms/итерация, память стабильна, ошибок в runs.md — 0
```

## Отладка

Если тест упал с ошибкой импорта:

```bash
# Убедись, что ты в корне проекта
cd /path/to/kiborg

# Запуск через python (не через python -m)
python stress/stress_test_harvest.py 5
```

Если память растёт бесконтрольно — смотри `tracemalloc` вывод в тесте (пики на каждой итерации). Возможные причины:
- Утечка в органах (например, кэш без лимита)
- Утечка в `seen_items` (без ротации)
- Утечка в `state.json` (растёт без bound)
